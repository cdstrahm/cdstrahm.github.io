Leverage Sports Data - leveragesportsdata.com

Tech: Next.js, TypeScript, Tailwind CSS, FastAPI, MySQL, Clerk, Stripe, VPS, pm2

  • Built and deployed a full-stack web app with a Next.js frontend and a FastAPI backend, both hosted on a Linux VPS and managed with pm2
  • Integrated Clerk for authentication and Stripe for payments, with all access control enforced at the API layer
  • Designed a component system with reusable chart, table, stat, and UI primitives to keep the dashboard consistent and extensible
  • Connected to a MySQL database via FastAPI to serve live player, game, and team data across multiple statistical dimensions

MLB Data Pipeline

The automated ETL pipeline that keeps the Leverage Sports Data database current. A nightly orchestration script runs 27 sync steps in dependency order, pulling data from external APIs and writing it to MySQL. Supports both incremental daily updates and full date-range backfills, and sends a Discord summary on completion.

Tech: Python, pandas, SQLAlchemy, MySQL, REST APIs, GitHub Actions

  • Designed the sync order around data dependencies so downstream tables always have the upstream rows they need
  • Built retry logic into each step (up to 3 attempts) to handle transient API failures without aborting the full run
  • Included backfill mode via CLI flags to re-sync historical date ranges without changing any pipeline logic
  • Wrote audit and validation scripts to diagnose missing records and verify data integrity independently of the sync

College Basketball Win Probability Model

A binary classification model that generates probabilistic win predictions for upcoming college basketball games using lagged team performance and efficiency metrics as features. Achieved 77% accuracy on a held-out test set of approximately 7,000 games. Outputs a predicted winner and win percentage for each matchup, with per-game feature contribution breakdowns available on request.

Tech: Python, XGBoost, scikit-learn, pandas, NumPy, MySQL

  • Engineered features using lagged game stats, rolling averages, and KenPom efficiency ratings, merging multiple data sources into a single training matrix
  • Enforced a strict chronological train/test split by date rather than by row to prevent temporal leakage
  • Extracted per-matchup feature contributions using XGBoost's prediction contribution output to explain individual predictions
  • Built a daily prediction pipeline that loads upcoming games, builds the feature set, drops incomplete records with logging, generates probabilities, and writes results to both CSV and the database

College Basketball Data Pipeline

An incremental ETL pipeline that keeps a MySQL database of college basketball data current. Pulls game results and team stats from the CBBd API and efficiency ratings from KenPom on a daily basis, resuming from the most recent record in the database each run. Supports full historical backfills via date range arguments.

Tech: Python, pandas, SQLAlchemy, MySQL, REST APIs

  • Designed incremental sync logic that resumes from the latest game date in the database, avoiding redundant fetches
  • Integrated two external data sources with different schemas into a unified relational structure
  • Built backfill mode with date range arguments to rebuild or repair any historical window without changing pipeline logic
  • Managed credentials and connection config through environment variables for consistent behavior across local and production environments
